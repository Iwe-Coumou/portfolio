Building a Static Site Generator from Scratch

2026-03-05

This portfolio is generated by a custom static site generator I built as part of the Boot.dev curriculum. Here's how it works.

Why Build One from Scratch

The assignment was clear: no shutil.copytree, no Hugo, no Jekyll. Build it yourself. The result is a deeper understanding of how Markdown parsers, HTML node trees, and template engines actually work.

The Pipeline

Every page goes through the same pipeline:

  1. Parse frontmatter to get metadata and template name
  2. Parse Markdown into a block tree
  3. Convert each block to an HTML node
  4. Load the template and insert partials
  5. Substitute all placeholders
  6. Write the output file

Markdown Parsing

The parser handles headings, paragraphs, lists, code blocks, blockquotes, and inline styles. Inline parsing is mutually recursive — parse_inline calls text_node_to_html_node which calls parse_inline again for nested styles like bold and italic.
